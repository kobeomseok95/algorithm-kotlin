# 알고리즘 풀이 커밋 & 푸시 커맨드

## 입력

사용자가 제출한 풀이 파일: `$ARGUMENTS`

## 절차

### 1. 파일 경로 파싱

파일 경로에서 플랫폼과 문제 번호를 추출한다.

- `src/leetcode/{difficulty}/{id}.kt` → 플랫폼: `leetcode`, 문제 번호: `{id}`
- `src/programmers/{level}/{id}.kt` → 플랫폼: `programmers`, 문제 번호: `{id}`

### 2. 풀이 코드 분석

파일을 읽고, 사용된 핵심 알고리즘/자료구조를 파악한다.

예시:
- 이분 탐색, DP, BFS, DFS, 그리디, 투 포인터, 슬라이딩 윈도우, 백트래킹, 유니온 파인드, 위상 정렬, 다익스트라, 트리, 해시맵, 스택, 큐, 힙, 정렬 등

### 3. 커밋 메시지 작성

```
{플랫폼} {문제번호} 풀이

{알고리즘} 알고리즘으로 풀이
```

예시:
```
leetcode 153 풀이

이분 탐색 알고리즘으로 풀이
```

### 4. 실행

1. 해당 파일을 `git add`한다
2. 위 형식으로 `git commit`한다
3. `git push`한다

## 규칙

- 커밋 대상은 인자로 받은 파일만 포함한다
- 알고리즘은 한국어로 작성한다
- 커밋 메시지 외에 불필요한 설명은 하지 않는다
- 완료 후 커밋 메시지만 간단히 출력한다
